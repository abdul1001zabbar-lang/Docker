1. What is Kubernetes?
Kubernetes is a container orchestration platform used to deploy, manage, scale, and automate containerized applications.
2. Why do we need Kubernetes?
High Availability
Auto Scaling
Self Healing
Load Balancing
Rolling Updates
3. What is a Cluster?
A Cluster is a collection of Nodes managed by Kubernetes.
4. What are the components of the Control Plane?
kube-apiserver
etcd
kube-scheduler
kube-controller-manager
5. What are Worker Node components?
kubelet
kube-proxy
Container Runtime
6. What is etcd?
A distributed key-value store that stores Kubernetes cluster data.
7. What is kubelet?
An agent running on each worker node that communicates with the API server.
8. What is kube-proxy?
Handles networking and load balancing for Services.
Pods
9. What is a Pod?
Smallest deployable unit in Kubernetes.
10. Can a Pod contain multiple containers?
Yes.
11. Why is a Pod ephemeral?
Pods can be destroyed and recreated at any time.
12. How do you create a Pod?
kubectl run nginx --image=nginx
13. List Pods
kubectl get pods
14. Describe a Pod
kubectl describe pod nginx
15. View Pod logs
kubectl logs nginx
ReplicaSet & Deployment
16. What is a ReplicaSet?
Maintains the desired number of Pod running at a time.
17. What is a Deployment?
Manages ReplicaSets and application updates.
18. Difference between Pod and Deployment?
Pod → Single instance
Deployment → Manages multiple Pods
19. Scale a Deployment
kubectl scale deployment nginx --replicas=5
20. Update Deployment image
kubectl set image deployment/nginx nginx=nginx:latest
21. Rollback Deployment
kubectl rollout undo deployment nginx
Services
22. Why do we need Services?
Pods have changing IP addresses.
23. Types of Services?
ClusterIP
NodePort
LoadBalancer
ExternalName
24. Default Service Type?
ClusterIP.
25. What is NodePort?
Exposes application on a node port.
26. What is LoadBalancer?
Exposes application using cloud load balancer.
27. What is Namespace?
Logical isolation within a cluster.
28. List Namespaces
kubectl get ns
29. Create Namespace
kubectl create ns dev
30. What is ConfigMap?
Stores non-sensitive configuration.
31. What is Secret?
Stores sensitive data like passwords.
32. Difference between ConfigMap and Secret?
ConfigMap → Plain text
Secret → Base64 encoded
Storage
33. What is Persistent Volume (PV)?
Cluster storage resource.
34. What is Persistent Volume Claim (PVC)?
Request for storage.
35. How does PVC bind to PV?
Based on:
Storage Class
Access Mode
Capacity
36. What happens if Pod is deleted?
Data remains if using PV/PVC.

Networking
37. What is Ingress?

Manages external HTTP/HTTPS access.

38. What is CNI?
Container Network Interface plugin.
CNI is a networking plugin that gives Pods IP addresses and enables communication between Pods.
Calico
Flannel
Cilium
39. What is DNS in Kubernetes?
Provides service discovery.

Scaling
40. What is HPA?
Horizontal Pod Autoscaler.

41. What metrics does HPA use?
CPU
Memory
Custom Metrics
42. Create HPA
kubectl autoscale deployment nginx --cpu-percent=50 --min=1 --max=10
